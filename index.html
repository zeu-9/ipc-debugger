IPC Debugger

Inter-Process Communication Simulation Debugger

CSE-316 CA2  ·  Simulation-Based Tool

i This is a simulation-based debugger. It models OS-level IPC (pipes, message queues, shared memory) in JavaScript — not actual POSIX calls. All deadlock detection, race condition analysis, and synchronization warnings are algorithm-driven simulations.
Automatic Mode
Algorithm-driven analysis with real-time auto-detection of deadlocks, race conditions, mutex contention, bottlenecks and synchronization issues — no manual triggers needed.
  • Auto deadlock detection (Banker's algorithm)
  • Race condition & synchronization warnings
  • Semaphore / mutex visualization
  • Throughput & bottleneck analysis
  • Improper sync detection

Manual Mode
Full control over all IPC parameters. Manually inject faults, configure synchronization primitives, and observe how each decision affects the system.
  • Manual deadlock / bottleneck injection
